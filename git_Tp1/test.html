What is Git, and what is it used for?
git is a distributed version control system. meaning that every developer's working copy of the code is also a repository that can contain the full history of all changes:This includes tracking modifications, additions, and deletions of files.
What is a repository in Git, and what is it used for?
un dépôt Git (repository) est un espace où sont stockées toutes les versions d'un projet, ainsi que son historique de modifications. on le utilise pour suivre les changements et maintenir une version organisée et contrôlée de notre code.

How do you merge branches in Git, and what is the purpose of merging?
utilisez la commande git merge [nom de la branche]Cela permet d'incorporer les changements effectués dans une branche de développement (par exemple, une fonctionnalité) dans la branche principale du projet.
How do you resolve conflicts in Git when merging branches?
Lorsqu'il y a des conflits entre les modifications de deux branches fusionnées, Git marque ces conflits dans les fichiers concernés. Les développeurs doivent résoudre ces conflits manuellement en modifiant les fichiers en question. Une fois les conflits résolus, ils peuvent finaliser la fusion en utilisant la commande git merge --continue.
